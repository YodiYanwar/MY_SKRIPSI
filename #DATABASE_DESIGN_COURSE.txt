Normalisasi Database	: Pendekatan sistematis untuk menimimalkan redundansi data agar database dapat bekerja dengan optimal

Tujuan			: 1. Mengurangi dan menghilangkan redudansi data 
			  2. Memastikan dependensi data (data berada pada table yan tepat)


https://www.lucidchart.com/pages/database-diagram/database-design

Struktur database dikakatan bagus/baik ketika :
	1. Menghemat ruang penyimpanan dengan menguranggi redundansi data
	2. Menjaga akurasi dan integrasi data

Proses mendesain database :
	1. Requirement Analisys, mengidentifikasi maksud & tujuan dibuatnya database
	2. Merangkai data kedalam table
	3. Menentukan primary key dan menganalisa relasi
